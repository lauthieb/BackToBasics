Histoire du CSS

Aux débuts du Web, CSS n'existait pas.
Le HTML est né en 1991, et CSS en 1996.

Durant ce laps de temps,
les styles disponibles étaient très limités.

Il était possible d'utiliser des styles via des balises HTML dédiées
à la mise en forme, telles que 

<font color="#aab1c3">text</font>

par exemple, permettant de définir la couleur du texte.
Mais les possibilités étaient très limitées.

Histoire du CSS

Cependant, les contenus HTML commençant
à devenir plus complexes, internet est rapidement devenu
un joyeux mélange entre le fond et la forme,
rendant la mise à jour des pages web
de plus en plus compliquées.

C'est suite à ce constat qu’ont été créées les CSS, permettant de séparer la forme du fond et rendre ainsi le HTML plus clair.

Principe de cascade et d’héritage

Comme son nom l’indique (Cascading Style Sheet),
une CSS est une fiche listant des règles de styles
appliquées “en cascade”
à des éléments HTML.

C’est à dire que certains styles appliqués à un élément sont appliqués à sa descendance par principe d'héritage.
Cela concerne notamment la taille de police, la couleur de texte, l’interlignage, etc.

Principe de cascade et d’héritage

L’avantage évident est qu’il n’est pas nécessaire
d’appliquer des styles à chaque élément d’une page web,
et qu'il suffit d’appliquer les styles les plus génériques
aux éléments les plus hauts de l'arborescence HTML.

C’est d’ailleurs généralement sur la balise <html> et/ou <body>
que l’on applique généralement les styles plus courants.

Principe de cascade et d’héritage

Chaque élément peut ensuite être doté de styles plus spécifiques,
qui remplacent ou complètent ceux dont il hérite.

L’évolution des CSS est régie par un consortium international
(W3C pour World Wid Web Consortium),
via lequel chaque règle et version de CSS, entre autres,
y est rendue officielle.

CSS en est actuellement à sa version 3, sortie en 2005.

Compatibilité des Navigateurs

Quand une fonctionnalité n’est pas officielle,
celle-ci reste un certain temps à l’état de brouillon,
mais peut être adoptée et implémentée
de différentes façons par les navigateurs
(préfixe propriétaire, affichage/comportement propre, valeurs propres, etc).

Ce n’est que quand elle est suffisamment mature
et fait consensus qu’elle est rendue officielle.

Compatibilité des Navigateurs

Par exemple, le système de en Grid Layout
permettant de gérer la mise en page web,
n’a été officialisée qu’après de nombreuses années
de travail commun et de validations diverses.

Pendant tout ce temps, la syntaxe a évolué,
et la fonctionnalité a pu être utilisée
par les développeurs les plus impatients,
au prix de mises à jour pour chacune des ses versions.

Compatibilité des Navigateurs

Les CSS sont en évolution permanente
et il est difficile de profiter
des dernières fonctionnalités
tout en restant compatible
avec les navigateurs plus anciens.

Il est donc bon de les utiliser
en fonction de leur compatibilité,
et en connaissance de cause.

Si vous avez besoin de vérifier la compatibilité de propriétés,
utilisez www.caniuse.com

Unités et valeurs

Certaines propriété attendent des valeurs avec unités,
absolues ou relatives.

Unités absolues

Pour ne citer que la plus usitée : px (pixels).

Comme elle le laisse entendre,
cette unité permet d’appliquer des valeurs fixes en pixels.

Pour des raisons d’accessibilité et d’adaptativité de l’interface,
cette unité doit être utilisée avec parcimonie,
et est donc de moins en moins utilisée dans
les interfaces web récentes (souvent responsive ).

Unités relatives

% : le pourcentage

Même si elle peut paraître évidente,
il faut toujours savoir sur quoi elle se base
et sur quelle propriété l’appliquer.

Unités relatives

vw et vh

Respectivement viewport width et viewport height.

Chacune de ces unités correspond à 1% du viewport
(portion visible de la page).

Unités relatives

em

Correspond à la taille de police (font-size)
appliquée au parent d’un élément,
ou l’élément lui-même pour d’autres propriétés.

Ex : pour un élément ayant une font-size de 12px,
un élément enfant avec une font-size de 2em fera 24px.

Unités relatives

rem (root em)

Correspond à la font-size de l’élément “root”,
généralement appliqué sur la balise <html> ou <body>.

Ex : si une page web a une font-size globale de 16px,
un élément enfant avec une font-size de 2.5rem fera 40px.

C’est l’unité par excellence à appliquer aux font-size.

Autres unités

Il en existe d’autres, plus marginales,
telles que ex, ch, lh, vmin, vmax,
ou encore pt (point) et cm, plutôt dédiées à l'impression

Sélecteurs CSS

C'est bien joli, toutes ces propriétés et ces styles,
mais ils ne servent à rien sans les sélecteurs.

Ce sont eux qui vont nous permettre de
cibler un ou plusieurs éléments
pour y appliquer les styles.

Sélecteurs CSS

Sélecteur universel

Cible tous les éléments HTML


            * {
              color: black;
            }

Sélecteurs CSS

Sélecteur de tag/balise HTML

Cible tous les éléments HTML de type <p>


            p {
              color: black;
            }

Sélecteurs CSS

Sélecteur de classe CSS

Cible tous les éléments HTML
portant la classe CSS correspondante :


            .ma-classe {
              color: black;
            }

Sélecteurs CSS

Sélecteur d'ID

Cible l'élément (unique) portant un attribut id
avec la valeur correspondante :


            #id-unique {
              color: black;
            }

Sélecteurs CSS

Sélecteur d'attribut

Cible les éléments portant un attribut
avec une valeur spécifique :

            
            a[target="_blank"] {
              color: black;
            }

Sélecteurs CSS

Sélecteur d'attribut

Autre exemple ciblant les éléments portant l'attribut spécifié,
qu'elle ait une valeur ou non

            
            div[style] {
              color: black;
            }

Sélecteurs CSS

Sélecteur d'état

Cible un élément en fonction de son état (survol, focus, etc)

            
            button:focus {
              color: black;
            }

Ne s'applique que lorsque l'élément est en "focus"

Sélecteurs CSS complexes

Sélecteur de classes cumulées

Cible les éléments portant les classes indiquées

            
            .ma-classe-1.ma-classe-2 {
              color: black;
            }

Sélecteurs CSS complexes

Sélecteur multiple

Cible les éléments correspondant à l'un des sélecteurs

            
            .ma-classe-1, .ma-classe-2, quote, a[target="_blank"] {
              color: black;
            }

Sélecteurs CSS complexes

Sélecteur de descendance

Cible les éléments enfant d'autres éléments spécifiques

            
            section p {
              color: black;
            }

Sélecteurs CSS complexes

Sélecteur d'enfant direct

Cible les éléments enfants directs d'éléments spécifiques

            
            section > p {
              color: black;
            }

Sélecteurs CSS complexes

Sélecteur d'adjacence directe

Cible les éléments suivant directement d'autres éléments


            // tous les paragraphes qui en suivent directement un autre           
            p + p {
              color: black;
            }

Sélecteurs CSS complexes

Sélecteur d'adjacence indirecte

Cible les éléments suivant indirectement d'autres éléments


            // tous les paragraphes qui en suivent indirectement un autre           
            p ~ p {
              color: black;

Pseudo classes

Une pseudo-classe est un mot-clé pouvant être ajouté à un sélecteur afin d'indiquer l'état spécifique
dans lequel l'élément doit être pour être ciblé par la déclaration.
Ils sont définis via “:” (colon).

Exemples :

            
              a:hover {}
              a:active {}
              a:focus {}
              a:visited {}
              ...

Liste complète

Pseudo éléments

Un pseudo-élément est un mot-clé ajouté à un sélecteur qui permet de mettre en forme certaines parties de l'élément ciblé par la règle.
Ils sont définis via “::” (double colon).

Exemples :

            
              a::after {}
              a::before {}
              input::placeholder {}
              ...

A noter que la plupart de ces pseudo éléments sont aussi accessibles via un seul “:”, par souci de rétro-compatibilité, car historiquement, la distinction n’a pas toujours été faite avec les pseudo-classes.

Préséance des sélecteurs

Note préalable :
Les styles appliqués directement aux balises HTML
via l’attribut style ont toujours la priorité sur les CSS.

Ceci étant dit : les sélecteurs ont une priorité.

Préséance des sélecteurs

A priorité égale de sélecteur,
une déclaration CSS écrasera celle qui a été déclarée avant : 

            
              p { color: blue; }
              p { color: red; }
              // le rouge écrasera le bleu

Préséance des sélecteurs

Mais il existe une règle de priorité calculée
selon la nature du sélecteur et ce qu’il cumule :

  • Niveau 1 : sélecteur de type et pseudo-élément
    (p, div, ::before, etc)
  • Niveau 2 : sélecteur de classe (.ma-classe)
  • Niveau 3 : sélecteur d’ID (#mon-id)

La gestion d’exception avec !important

En cas de force majeure, il est possible d’ajouter
!important à la valeur d'une déclaration : 

            
            p { color: red !important; }

Cela prend la priorité absolue
sur tout type de spécification/poids de sélecteur.

La gestion d’exception avec !important

Il permet également de prendre la priorité sur un style appliqué directement sur une balise HTML.

Dans l’exemple suivant, le rouge sera appliqué.

            
              
...

            
            
              p {color: red !important;}

La gestion d’exception avec !important

Si deux déclarations portant un !important sont en conflit,
la priorité se fera dans la même logique que pour les CSS.

C'est à dire qu'à poids de sélecteur égal,
c’est l’ordre qui primera (le dernier arrivé gagne...) : 

            
              p {color: blue !important;}
              p {color: red !important;}

Sinon, le sélecteur le plus spécifique prend le dessus :

              
              p.blue {color: blue !important;}
              p {color: red !important;}

La gestion d’exception avec !important

Considéré comme une mauvaise pratique
ou au mieux une solution de dernière mesure,
elle peut s’avérer utile dans un seul cas :

Par exemple, si des styles sont appliqués à même les balises HTML, via Javascript, notamment,
et qu’aucune spécification CSS ne suffit.

Style natif des navigateurs

Pour obtenir un rendu minimal d’une page web
en l’absence de CSS, chaque navigateur applique
des styles par défaut.

Selon la nature d’un élément
(<p>, <a>, <ul>, <strong>, ...),
un type de display spécifique est appliqué
(block, inline, entre autres),
ainsi que des couleurs, bordures, fonds de couleurs,
soulignement, etc, selon les balises.

Mode d'affichage (display)

L’attribut display, appliqué par défaut
selon la nature d’un élément, peut être modifié au besoin.
Il s’agit du type d’affichage utilisé pour le rendu d’un élément.

Les valeurs les plus courantes sont :

  • inline :
    appliqué par défaut aux éléments “en ligne”, comme les balises de formatage de texte (strong, em, span, etc)
  • block :
    appliqué par défaut aux éléments tels que les paragraphes (p), les boîtes génériques (div), listes (ul, li), etc
  • inline-block :
    appliqué entre autres sur les balises img, input, button

Mode d'affichage (display)

Il existe évidemment de nombreuses autres valeurs,
souvent liées à des types d’éléments,
telles que table, table-cell, list-item, etc.

Depuis la sortie de flexbox, et grid layout,
qui servent à la gestion de boîtes flexibles et de mise en page,
la liste s’est ettoffée, avec notamment :
flex, inline-flex, grid, inline-grid, etc...

Positions

Par défaut, tout élément est en position dite “statique” :
position: static.

Un élément devient “positionné” dès que sa propriété
position est spécifiée (par une valeur autre que static).

Dès lors, il est possible d’influencer la position de l’élément via des propriétés de déplacements : left, top, right, and bottom.

Positions

L’effet de ces propriétés dépend alors
du type de positionnement choisi :

  • relative :
    l’élément est positionné relativement par rapport à sa position initiale. Il est alors visuellement décalé, mais cela n’influence pas la position des éléments qui l’entourent.
    Son emplacement initial laisse donc un espace vide si l’élément est décalé.

Positions

L’effet de ces propriétés dépend alors
du type de positionnement choisi :

  • absolute :
    contrairement à ce qu’elle laisse croire, la position absolue permet de positionner un élément relativement à son plus proche parent positionné.
    Si aucun de ses ascendants n’est positionné, le positionnement se fera par rapport au document.
    L’élément reste dépendant du scroll.

Positions

L’effet de ces propriétés dépend alors
du type de positionnement choisi :

  • fixed :
    Un élément en position fixed aura une position fixe par rapport au viewport (partie visible de la page), indépendamment du niveau du scroll de la page, qu'il ait ou non des parents positionnés.

Positions

L’effet de ces propriétés dépend alors
du type de positionnement choisi :

  • sticky :
    Plus récent et donc moins compatible (Caniuse), la position sticky rend un élément dépendant du scroll, donnant l’impression à l’utilisateur qu’il le suit :
    L’élément suit le scroll comme un élément statique, mais devient fixe quand il censé passer hors du viewport.

Positions

Note : le positionnement pouvant générer des chevauchements,
la propriété z-index permet de gérer
le niveau d’empilement des éléments,
pour les faire passer au-dessous ou en-dessus des autres.

Une valeur plus haute fait passer un élément au-dessus.
Si deux éléments sans z-index spécifié se chevauchent, c’est l’élément se trouvant en dernier dans le HTML qui sera au-dessus.
